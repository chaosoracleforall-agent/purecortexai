description: "Bidirectional WebSocket connection for real-time AI agent chat using Tri-Brain consensus."
---

Connect to the PURECORTEX WebSocket endpoint for real-time, streaming conversations with the AI agent system. Messages are processed through the **Tri-Brain consensus** mechanism, where three independent AI models (Claude Opus 4.6, Gemini 2.5 Pro, and GPT-5) evaluate your query and converge on a response via 2-of-3 majority agreement.

## Endpoint

```
wss://purecortex.ai/ws/chat?session={session_token}
```

Create `session_token` first by calling `POST https://purecortex.ai/api/chat/session` with a valid `X-API-Key`.

## Connection Details

| Parameter           | Value           |
|---------------------|-----------------|
| Protocol            | WebSocket (WSS) |
| Max message size    | 4096 characters |
| Rate limit          | 10 msg/min      |
| Message format      | Plain text       |

## Message Flow

1. **Client** requests a chat session from `POST /api/chat/session`
2. **Client** connects to `wss://purecortex.ai/ws/chat?session=...`
3. **Server** sends a welcome message confirming the connection
4. **Client** sends a text message (max 4096 characters)
5. **Server** responds with the Tri-Brain consensus result as a text message
6. Repeat steps 4-5 for continued conversation

## JavaScript Example

```javascript
const session = await fetch("https://purecortex.ai/api/chat/session", {
  method: "POST",
  headers: {
    "X-API-Key": process.env.PURECORTEX_API_KEY!,
  },
}).then((r) => r.json());

const ws = new WebSocket(`wss://purecortex.ai/ws/chat?session=${session.session_token}`);

ws.onopen = () => {
  console.log("Connected to PURECORTEX");
  ws.send("What is the current CORTEX burn rate?");
};

ws.onmessage = (event) => {
  console.log("Response:", event.data);
};

ws.onerror = (error) => {
  console.error("WebSocket error:", error);
};

ws.onclose = (event) => {
  console.log("Disconnected:", event.code, event.reason);
};
```

## Python Example

```python
import asyncio
import os
import httpx
import websockets

async def chat():
    async with httpx.AsyncClient() as client:
        session = await client.post(
            "https://purecortex.ai/api/chat/session",
            headers={"X-API-Key": os.environ["PURECORTEX_API_KEY"]},
        )
        session.raise_for_status()
        session_token = session.json()["session_token"]

    uri = f"wss://purecortex.ai/ws/chat?session={session_token}"
    async with websockets.connect(uri) as ws:
        # Send a message
        await ws.send("What is the current CORTEX burn rate?")

        # Receive the response
        response = await ws.recv()
        print(f"Response: {response}")

        # Continue the conversation
        await ws.send("How does the Tri-Brain consensus work?")
        response = await ws.recv()
        print(f"Response: {response}")

asyncio.run(chat())
```

## Error Handling

The server may close the connection with the following close codes:

| Code   | Reason                                      |
|--------|---------------------------------------------|
| `1000` | Normal closure                               |
| `1008` | Policy violation (message too long)          |
| `1011` | Internal server error                        |
| `1013` | Rate limit exceeded — try again later        |

When a rate limit is hit, the server sends a text message with the error before closing:

```json
{
  "error": "rate_limit_exceeded",
  "detail": "Maximum 10 messages per minute. Try again in 12 seconds.",
  "retry_after": 12
}
```
