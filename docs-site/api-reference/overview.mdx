## Base URL

All API requests are made to:

```
https://purecortex.ai
```

## Authentication

Read-only endpoints (transparency, governance, agents) are **public** and require no authentication.

Protected REST endpoints require an `X-API-Key`. WebSocket chat requires a short-lived session token minted by `POST /api/chat/session`.

## Recommended Entry Points

- Python SDK: `pip install ./sdk/python`
- TypeScript SDK: `npm install ./sdk/typescript`
- CLI: `pip install ./cli`

These first-party clients map directly to the live endpoint surface and are the preferred way to integrate unless you specifically need raw HTTP.

<Info>
  The public docs should be treated as the current testnet surface, not a promise of future mainnet features. Where behavior differs during rollout, prefer the first-party clients or the live OpenAPI schema generated from the backend.
</Info>

## Endpoint Families

| Family | Endpoints |
|--------|-----------|
| Health | `/health` |
| Marketplace | `/api/marketplace/config`, `/api/marketplace/agents/{asset_id}/state`, `/api/marketplace/quote/buy`, `/api/marketplace/quote/sell` |
| Transparency | `/api/transparency/*` |
| Agents | `/api/agents/registry`, `/api/agents/{agent}/activity`, `/api/agents/{agent}/chat` |
| Governance | `/api/governance/*` |
| Chat bootstrap | `/api/chat/session` |
| WebSocket | `/ws/chat?session=...` |

## Rate Limits

| Channel        | Limit           |
|----------------|-----------------|
| REST API       | 60 req/min      |
| WebSocket      | Runtime-enforced burst control |
| Transparency   | 120 req/min     |

Rate-limited HTTP responses return `429`. WebSocket chat enforces its own runtime message pacing.

## Response Format

All endpoints return JSON with **snake_case** field names.

Successful responses return the data directly:

```json
{
  "status": "ok",
  "version": "0.7.7"
}
```

Error responses typically include a `detail` field:

```json
{
  "detail": "Proposal with the given ID does not exist."
}
```

## Typical Chat Flow

```text
1. Create or load a long-lived API key
2. POST /api/chat/session with X-API-Key
3. Connect to wss://purecortex.ai/ws/chat?session=<token>
4. Exchange plain text messages
```

Signed governance voting follows a separate path:

```text
1. Load governance proposals from /api/governance/proposals
2. Check voting power with GET /api/governance/proposals/{proposal_id}/power/{address}
3. Sign the canonical vote message in-wallet
4. POST /api/governance/proposals/{proposal_id}/vote-signed
```

## Error Codes

| HTTP Code | Meaning                | Description                                          |
|-----------|------------------------|------------------------------------------------------|
| `200`     | OK                     | Request succeeded.                                   |
| `400`     | Bad Request            | Malformed request body or missing required fields.   |
| `401`     | Unauthorized           | Missing or invalid API key or chat session. |
| `404`     | Not Found              | The requested resource does not exist.               |
| `429`     | Too Many Requests      | Rate limit exceeded. Retry after the indicated delay. |
| `500`     | Internal Server Error  | Unexpected server-side failure.                      |
| `503`     | Service Unavailable    | The service is temporarily down for maintenance.     |
