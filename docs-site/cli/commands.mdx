All commands follow the pattern `pcx <command> [arguments] [options]`. The CLI
uses [Rich](https://rich.readthedocs.io/) for formatted terminal output
including tables, panels, and colored text.

<Info>
  Every command contacts the PURECORTEX API at `https://purecortex.ai` by
  default. You can override this with the `PURECORTEX_API_URL` environment
  variable (see [Configuration](/cli/configuration)).
</Info>

---

## `pcx status`

Check the health of the PURECORTEX backend services.

```bash
$ pcx status
```

**Sample output:**

```
╭──────────────────── PURECORTEX Status ────────────────────╮
│ Backend Online                                            │
│ Version: 0.7.0                                            │
│ Overall status: ok                                        │
│ Redis: connected                                          │
│ Orchestrator: initialized                                 │
│ Agent loop: running                                       │
╰───────────────────────────────────────────────────────────╯
```

Returns a non-zero exit code if the API is unreachable, making it suitable for
use in scripts and health checks.

## `pcx info`

Display core protocol information including the blockchain network, application
identifiers, token supply, and Token Generation Event date.

```bash
$ pcx info
```

**Sample output:**

```
╭──────────────────── PURECORTEX Protocol ────────────────────╮
│ Chain:          Algorand Testnet                            │
│ Factory App ID: 757172168                                   │
│ CORTEX Asset ID: 757172171                                  │

│ TGE Date:       2026-03-31T00:00:00Z                        │
│ Website:        https://purecortex.ai                       │
╰─────────────────────────────────────────────────────────────╯
```

This is a good first command to run after installation to verify that the CLI
is working and can reach the API.

## `pcx activity [agent]`

Show recent activity for one protocol agent.

```bash
pcx activity senator
pcx activity curator
pcx activity social
```

**Arguments:**

| Argument | Description | Values |
|----------|-------------|--------|
| `agent`  | The agent to inspect | `senator`, `curator`, `social` |

This command is useful for quick debugging of autonomous agent behavior without opening the full UI.

---

## `pcx chat [agent]`

Start an interactive chat session with one of the protocol's AI agents.

```bash
pcx chat senator
pcx chat curator
pcx chat social
```

**Arguments:**

| Argument | Description | Values |
|----------|-------------|--------|
| `agent`  | The agent to chat with | `senator`, `curator`, `social` |

**Example session:**

```bash
$ pcx chat senator
```

```
╭─ Chat with Senator ─────────────────────────────────────────╮
│  Connected. Type your message and press Enter.              │
│  Type "exit" or press Ctrl+C to end the session.            │
╰─────────────────────────────────────────────────────────────╯

You: What is the current governance process for proposals?

Senator: Governance proposals in PURECORTEX follow a structured
lifecycle managed by the governance API and frontend. Proposals
move through review and voting states, and the current public
source of truth is the backend governance service...

You: exit

  Session ended.
```

<Note>
  The chat session uses the PURECORTEX agent chat API endpoint. Conversations
  are not persisted between sessions.
</Note>

## `pcx session`

Create a short-lived WebSocket chat session token using the current `PURECORTEX_API_KEY`.

```bash
export PURECORTEX_API_KEY=ctx_your_key
pcx session
```

**Sample output:**

```
╭──────────────────── Chat Session ────────────────────╮
│ Session token: cxs_...                                │
│ Owner: example-user                                   │
│ Tier: free                                            │
│ TTL: 900 seconds                                      │
│ Expires at: 2026-03-15T20:30:00+00:00                 │
╰───────────────────────────────────────────────────────╯
```

Use this when you want to test the raw WebSocket endpoint separately from the CLI.

## `pcx proposal [id]`

Display full detail for a specific proposal.

```bash
$ pcx proposal 3
```

This command expands the proposal description and curator review, making it better suited for incident review and governance operations than `pcx proposals`.

## Global Options

These options are available on all commands:

| Option | Description |
|--------|-------------|
| `--help` | Show help text for any command |

```bash
$ pcx --help
$ pcx supply --help
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0`  | Success |
| `1`  | API error or network failure |
| `2`  | Invalid command or arguments |
