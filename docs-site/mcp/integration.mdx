# Integration Guide

This guide covers the current supported integration path: local connection via Claude Desktop or another stdio-capable MCP client.

---

## Claude Desktop

Claude Desktop supports MCP servers natively. Add PURECORTEX to your configuration file to make all three tools available inside Claude conversations.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/chaosoracleforall-agent/purecortexai.git
    cd purecortexai/backend
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    ```
  </Step>
  <Step title="Configure Claude Desktop">
    Open your Claude Desktop configuration file:

    <Tabs>
      <Tab title="macOS">
        ```
        ~/Library/Application Support/Claude/claude_desktop_config.json
        ```
      </Tab>
      <Tab title="Windows">
        ```
        %APPDATA%\Claude\claude_desktop_config.json
        ```
      </Tab>
      <Tab title="Linux">
        ```
        ~/.config/Claude/claude_desktop_config.json
        ```
      </Tab>
    </Tabs>

    Add the PURECORTEX server to the `mcpServers` object:

    ```json
    {
      "mcpServers": {
        "purecortex": {
          "command": "python",
          "args": ["/absolute/path/to/purecortexai/backend/mcp_server.py"],
          "env": {
            "CLAUDE_API_KEY": "your-claude-api-key",
            "GEMINI_API_KEY": "your-gemini-api-key"
          }
        }
      }
    }
    ```
  </Step>
  <Step title="Restart Claude Desktop">
    Restart the application. You should see PURECORTEX listed in the MCP tools panel. You can now ask Claude to invoke tools like `get_tri_brain_consensus`.
  </Step>
</Steps>

<Warning>
  The `CLAUDE_API_KEY` and `GEMINI_API_KEY` environment variables are required for the tri-brain orchestrator. Without them, the `get_tri_brain_consensus` tool will return an initialization error. The keys are loaded from environment variables first, then from GCP Secret Manager as a fallback.
</Warning>

## Custom Agent Integration

If you are building an autonomous agent that needs to consume PURECORTEX intelligence locally, use the MCP SDK for your language:

<CardGroup cols={2}>
  <Card title="Python SDK" icon="python" href="https://github.com/modelcontextprotocol/python-sdk">
    `pip install mcp`
  </Card>
  <Card title="TypeScript SDK" icon="js" href="https://github.com/modelcontextprotocol/typescript-sdk">
    `npm install @modelcontextprotocol/sdk`
  </Card>
</CardGroup>

### Minimal Agent Example

```python
import asyncio
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def autonomous_loop():
    """Agent that queries PURECORTEX for consensus every hour."""
    params = StdioServerParameters(
        command="python",
        args=["/absolute/path/to/purecortexai/backend/mcp_server.py"],
    )

    async with stdio_client(params) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()

            while True:
                result = await session.call_tool(
                    "get_tri_brain_consensus",
                    {"prompt": "Should I rebalance my portfolio given current conditions?"}
                )
                response = result.content[0].text
                print(f"PURECORTEX says: {response}")

                # Act on the response...
                await asyncio.sleep(3600)

asyncio.run(autonomous_loop())
```

## Composability Scoring

When your agent participates in the PURECORTEX MCP ecosystem, future registry and composability features may track how frequently and reliably tools are used across the network.

<AccordionGroup>
  <Accordion title="How scoring works">
    The current repository documents composability as a protocol direction rather than a fully deployed public feature.
  </Accordion>
  <Accordion title="Score factors">
    - Invocation count
    - Success rate
    - Unique callers
    - Uptime
  </Accordion>
  <Accordion title="Incentives">
    Reward and scoring mechanics should not be treated as live unless they are explicitly enabled in the active deployment.
  </Accordion>
</AccordionGroup>

<Info>
  The tracked repository currently supports local stdio MCP usage. Do not assume a public remote MCP endpoint or live composability registry unless the deployment docs explicitly add one.
</Info>
