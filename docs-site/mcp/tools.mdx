# Available Tools

The current PURECORTEX MCP server exposes one consensus tool plus several practical read-only tools backed by the public API. All tool calls are validated by the `PermissionProxy` before execution. Any action that exceeds the server's permission tier is blocked and returns a security error.

<Warning>
  The MCP server runs at the `READ_ONLY` permission tier by default. Tool responses that would require elevated permissions (e.g., triggering an on-chain swap) are rejected by the sandbox.
</Warning>

---

## `get_tri_brain_consensus`

Submit a prompt for parallel inference across Claude Opus 4.6, Gemini 2.5 Pro, and GPT-5. The protocol evaluates 2-of-3 majority consensus and returns a validated response.

<ParamField body="prompt" type="string" required>
  The natural language prompt to evaluate. This is sent to all three models in parallel. Input is automatically tagged with security boundaries to prevent prompt injection.
</ParamField>

### Response

<ResponseField name="ok" type="boolean">
  `true` when the request completed successfully.
</ResponseField>

<ResponseField name="action" type="string">
  The validated consensus action type, usually `RESPOND` or `REPLY`.
</ResponseField>

<ResponseField name="message" type="string">
  The consensus response content.
</ResponseField>

<ResponseField name="decision" type="object">
  The raw validated decision object returned by the orchestrator.
</ResponseField>

### Example

<CodeGroup>

```python Python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

params = StdioServerParameters(command="python", args=["mcp_server.py"])

async with stdio_client(params) as (read, write):
    async with ClientSession(read, write) as session:
        await session.initialize()

        result = await session.call_tool(
            "get_tri_brain_consensus",
            {"prompt": "What is the current state of DeFi on Algorand?"}
        )
        print(result.content[0].text)
```

```typescript TypeScript

    prompt: "What is the current state of DeFi on Algorand?"
  }
```

## Public Read-Only Tools

These tools are backed by the live public API and are intended for dashboards, IDE assistants, autonomous agents, and protocol observers that need structured reads without writing their own HTTP client layer.

### `get_protocol_health`

Return the current health payload from `GET /health`.

### `get_agent_registry`

Return the public registry of protocol agents from `GET /api/agents/registry`.

### `get_agent_activity`

Fetch recent activity for a single agent.

<ParamField body="agent_name" type="string" required>
  One of `senator`, `curator`, or `social`.
</ParamField>

### `get_governance_overview`

Fetch the governance counters from `GET /api/governance/overview`.

### `list_governance_proposals`

Fetch the proposal list and optionally trim it to the newest `N` entries.

<ParamField body="limit" type="integer">
  Maximum number of proposals to return. Defaults to `10`.
</ParamField>

### `get_governance_proposal`

Fetch a single proposal by numeric identifier.

<ParamField body="proposal_id" type="integer" required>
  Governance proposal ID.
</ParamField>

### `get_transparency_snapshot`

Return a bundled snapshot of supply, treasury, burns, and governance transparency data.

### Example

```python Python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

params = StdioServerParameters(command="python", args=["mcp_server.py"])

async with stdio_client(params) as (read, write):
    async with ClientSession(read, write) as session:
        await session.initialize()

        health = await session.call_tool("get_protocol_health", {})
        proposals = await session.call_tool("list_governance_proposals", {"limit": 5})
        print(health.content[0].text)
        print(proposals.content[0].text)
```

## Security Model

All MCP tool calls pass through the `PermissionProxy` before execution:

<Steps>
  <Step title="Request Received">
    The MCP server receives a JSON-RPC `tools/call` request from the client.
  </Step>
  <Step title="Tri-Brain Inference">
    For `get_tri_brain_consensus`, the prompt is sent to Claude Opus 4.6, Gemini 2.5 Pro, and GPT-5 in parallel. User input is wrapped in `<user_query>` tags with explicit security boundary markers.
  </Step>
  <Step title="Permission Validation">
    The `PermissionProxy` checks whether the consensus action is permitted at the server's current tier. Actions like `REPLY` and `RESPOND` pass at `READ_ONLY`. Actions like `SWAP` or `EXECUTE` require `TREASURY_SWAP` tier and are blocked.
  </Step>
  <Step title="Response Returned">
    If validation passes, the tool result is returned to the MCP client. If blocked, the server returns: `"Action blocked by PURECORTEX Security Proxy."`
  </Step>
</Steps>

| Permission Tier | Allowed Actions |
|----------------|----------------|
| `READ_ONLY` | `REPLY`, `RESPOND`, `MONITOR`, `ANALYZE` |
| `SOCIAL_POST` | Above + `POST` |
| `ASSET_MANAGEMENT` | Above + `PROPOSE`, `APPROVE`, `REJECT`, `CREATE_AGENT` |
| `TREASURY_SWAP` | Above + `SWAP`, `EXECUTE` |
