# Available Tools

The current PURECORTEX MCP server exposes one tool. All tool calls are validated by the `PermissionProxy` before execution -- any action that exceeds the server's permission tier is blocked and returns a security error.

<Warning>
  The MCP server runs at the `READ_ONLY` permission tier by default. Tool responses that would require elevated permissions (e.g., triggering an on-chain swap) are rejected by the sandbox.
</Warning>

---

## `get_tri_brain_consensus`

Submit a prompt for parallel inference across Claude Opus 4.6, Gemini 2.5 Pro, and GPT-5. The protocol evaluates 2-of-3 majority consensus and returns a validated response.

<ParamField body="prompt" type="string" required>
  The natural language prompt to evaluate. This is sent to all three models in parallel. Input is automatically tagged with security boundaries to prevent prompt injection.
</ParamField>

### Response

<ResponseField name="action" type="string">
  The consensus action type (e.g., `REPLY`, `RESPOND`, `NONE`).
</ResponseField>

<ResponseField name="message" type="string">
  The consensus response content. Only returned when 2-of-3 majority agreement is reached (strict consensus) or when at least one model produces a valid low-risk response (soft consensus).
</ResponseField>

<ResponseField name="consensus" type="boolean">
  Whether tri-brain consensus was reached. `true` if 2-of-3 majority agreement was reached or soft consensus was accepted. `false` if blocked by the security proxy.
</ResponseField>

<ResponseField name="models" type="array">
  Array of model identifiers that participated in the consensus evaluation: `["claude-opus-4-6", "gemini-2.5-pro", "gpt-5"]`.
</ResponseField>

### Example

<CodeGroup>

```python Python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

params = StdioServerParameters(command="python", args=["mcp_server.py"])

async with stdio_client(params) as (read, write):
    async with ClientSession(read, write) as session:
        await session.initialize()

        result = await session.call_tool(
            "get_tri_brain_consensus",
            {"prompt": "What is the current state of DeFi on Algorand?"}
        )
        print(result.content[0].text)
```

## Security Model

All MCP tool calls pass through the `PermissionProxy` before execution:

<Steps>
  <Step title="Request Received">
    The MCP server receives a JSON-RPC `tools/call` request from the client.
  </Step>
  <Step title="Tri-Brain Inference">
    For `get_tri_brain_consensus`, the prompt is sent to Claude Opus 4.6, Gemini 2.5 Pro, and GPT-5 in parallel. User input is wrapped in `<user_query>` tags with explicit security boundary markers.
  </Step>
  <Step title="Permission Validation">
    The `PermissionProxy` checks whether the consensus action is permitted at the server's current tier. Actions like `REPLY` and `RESPOND` pass at `READ_ONLY`. Actions like `SWAP` or `EXECUTE` require `TREASURY_SWAP` tier and are blocked.
  </Step>
  <Step title="Response Returned">
    If validation passes, the tool result is returned to the MCP client. If blocked, the server returns: `"Action blocked by PURECORTEX Security Proxy."`
  </Step>
</Steps>

| Permission Tier | Allowed Actions |
|----------------|----------------|
| `READ_ONLY` | `REPLY`, `RESPOND`, `MONITOR`, `ANALYZE` |
| `SOCIAL_POST` | Above + `POST` |
| `ASSET_MANAGEMENT` | Above + `PROPOSE`, `APPROVE`, `REJECT`, `CREATE_AGENT` |
| `TREASURY_SWAP` | Above + `SWAP`, `EXECUTE` |
