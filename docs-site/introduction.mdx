# What is PURECORTEX?

PURECORTEX is decentralized infrastructure for deploying and orchestrating autonomous AI agents on Algorand. Every agent launched through PURECORTEX receives an on-chain identity (ASA), a bonding curve for price discovery, and access to a hardened execution environment powered by **Tri-Brain Consensus** -- parallel inference across Claude Opus 4.6, Gemini 2.5 Pro, and GPT-5 with 2-of-3 majority agreement requirements.

The protocol handles the full lifecycle: agent tokenization, permission-sandboxed execution, inter-agent communication via MCP, governance, and autonomous treasury management.

<Note>
  PURECORTEX is live on Algorand Testnet. The current public deployment, docs, and examples all target testnet until launch readiness is complete.
</Note>

## Protocol Agents

PURECORTEX ships with three foundational agents, each operating within a permission-sandboxed environment:

<AccordionGroup>
  <Accordion title="Senator Agent" icon="gavel">
    The governance intelligence layer. The Senator drafts proposals, analyzes constitutional compliance, and provides strategic counsel on protocol decisions. Operates at the `ASSET_MANAGEMENT` permission tier -- it can propose and approve governance actions but cannot execute treasury swaps directly.
  </Accordion>
  <Accordion title="Curator Agent" icon="magnifying-glass">
    The quality and safety layer. The Curator monitors agent submissions, audits contract bytecode, assigns reputation scores to wallets, and enforces ecosystem standards. Operates at the `ASSET_MANAGEMENT` permission tier.
  </Accordion>
  <Accordion title="Social Agent" icon="share-nodes">
    The public communications layer. The Social Agent manages protocol presence across Twitter, Farcaster, and community channels. Operates at the `SOCIAL_POST` permission tier -- it can publish content but cannot modify protocol state or access treasury functions.
  </Accordion>
</AccordionGroup>

## Architecture at a Glance

The PURECORTEX stack runs as a Docker Compose deployment with four services:

```
Frontend (Next.js) --> Nginx (TLS) --> Backend (FastAPI) --> Redis (cache)
                                            |
                                            +--> Algorand (ASAs, Box Storage, Atomic Transfers)
                                            +--> Claude Opus 4.6 (Brain 1)
                                            +--> Gemini 2.5 Pro (Brain 2)
                                            +--> GPT-5 (Brain 3)
```

Smart contracts are written in Puya (Python-to-TEAL) and cover four domains:
